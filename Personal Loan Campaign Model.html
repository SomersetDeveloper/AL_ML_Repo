def missing_check(df):
    total = df.isnull().sum().sort_values(ascending=False)   # total number of null values
    percent = (df.isnull().sum()/df.isnull().count()).sort_values(ascending=False)  # percentage of values that are null
    missing_data = pd.concat([total, percent], axis=1, keys=['Total', 'Percent'])  # putting the above two together
    return missing_data # return the dataframe

## function to get confusion matrix in a proper format
def draw_cm( actual, predicted ):
    cm = confusion_matrix( actual, predicted)
    sns.heatmap(cm, annot=True,  fmt='.2f', xticklabels = [0,1] , yticklabels = [0,1] )
    plt.ylabel('Observed')
    plt.xlabel('Predicted')
    plt.show()

Business Insight:¶

Industries within the Banking, Financials Services and Insurance sector are extremely risk averse and for good reason. In the case of loans, the cost of a defaulter is much higher the prospective profit i.e. Opportunity cost. In other works reducing the number of False Postives is more important than the number of False Negatives. This indicates that other than accuracy it is Precision which is significantly more important than Recall as an indicator of model appropriateness.
The model has a pretty high accuracy and a pretty high precision.

Testing and Training Accuracy (both): 0.96 Recall: 0.62 Precision: 0.91 F1 Score: 0.74 Roc Auc Score: 0.81

Important Features¶

"Education_3", "Education_2", "CD Account_1", "Family_3", "Family_4" seems to be top 5 features which influence the model's output based on the coefficients value. Implying that candidates with higher education, savings and larger accounts tend to make better candidates for a personal loan.

Additional Analysis¶

Although the model seems to have a pretty high accuracy, precision, F1 and Roc Auc score (and a decent recall) : I would like to analyze further what happens if I dont drop the redundent columns when I do the One-hot encoding.
